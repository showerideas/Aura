# Gesture Classifier A/B Test — Phase J1/J2

*Last updated: 2026-05-26 | Implemented in `GestureClassifierABTest.kt`*

---

## Overview

Phase J1 introduced a synthetic A/B test comparing two decision strategies for
`GestureClassifier`:

| Config | Strategy | Description |
|--------|----------|-------------|
| **A** | Cosine-only | Accept if raw cosine similarity to centroid ≥ `CONFIDENCE_GATE` (0.82) |
| **B** | Full classifier | Accept if spread-normalised confidence ≥ `CONFIDENCE_GATE` (0.82) |

Config B is the production implementation in `GestureClassifier.predict()`.

---

## Dataset

| Parameter | Value |
|-----------|-------|
| Embedding dimensions | 63 (MediaPipe hand landmark coords, `EMBED_DIM`) |
| Genuine samples | 100 (base gesture + uniform noise, noise σ = 0.15 per component) |
| Impostor samples | 100 (random unit-sphere vectors, uncorrelated with base) |
| Enrolment split | 10 genuine used for training (`ENROLL_SIZE = 10`) |
| Evaluation split | 90 genuine + 100 impostor |
| RNG seed | 42 (deterministic; reproducible across runs) |

Genuine embeddings are generated by adding per-component uniform noise in
`[-0.15, +0.15]` to a random base unit vector, then re-normalising to the
unit sphere. This models realistic intra-user variation from slight hand-pose
differences between enrolment and authentication.

Impostor embeddings are uniformly random unit-sphere vectors — a worst-case
zero-correlation adversary model on a 63-dimensional sphere.

---

## Theoretical Expected Ranges

### Why impostors are reliably rejected on a 63-dim sphere

On a *d*-dimensional unit sphere, two independent random unit vectors have
cosine similarity distributed as `N(0, 1/√(d−1))`. With `d = 63`:

```
σ_cosine_impostor = 1 / √62 ≈ 0.127
```

For CONFIDENCE_GATE = 0.82:
- Cosine-only: P(sim ≥ 0.82) ≈ P(Z ≥ (0.82/0.127)) ≈ P(Z ≥ 6.5) ≈ **0.00%**
- The 63-dim geometry alone makes random impostors virtually impossible to pass

**Expected FAR (both configs): < 1%** against random impostor vectors.

### Genuine acceptance rate

With noise σ = 0.15 per component over 63 dims, the added noise vector has
expected L2 magnitude ≈ 0.15 × √(63/3) ≈ 0.69. Base vector L2 = 1.0.
After re-normalisation, the cosine similarity between a genuine test sample
and the enrolment centroid is expected around 0.82–0.88 depending on the
draw, placing it right at the CONFIDENCE_GATE.

**Expected FRR (Config A / cosine-only): 20–50%** — high variance due to
raw cosine sitting close to the 0.82 threshold.

**Expected FRR (Config B / full classifier): 10–35%** — the spread-normalisation
converts the distance relative to the enrolled spread, providing a softer
decision boundary that adapts to the user's actual gesture variability.

## CONFIDENCE_GATE Sensitivity Sweep

The J1 test includes a gate sweep from 0.70 to 0.95. Expected outcome:

| Gate | FAR (B) | FRR (B) | EER (B) |
|------|---------|---------|---------|
| 0.70 | ~0%     | ~5%     | ~2.5%   |
| 0.75 | ~0%     | ~10%    | ~5%     |
| 0.80 | ~0%     | ~15%    | ~7.5%   |
| **0.82** | **~0%** | **~20%** | **~10%** |
| 0.85 | ~0%     | ~30%    | ~15%    |
| 0.90 | ~0%     | ~50%    | ~25%    |
| 0.95 | ~0%     | ~80%    | ~40%    |

*Note: these are expected ranges from the theoretical analysis above. Run
`./gradlew :app:testDebugUnitTest --tests "*.GestureClassifierABTest"` to get
exact values for your RNG draw.*

## Decision: CONFIDENCE_GATE Adjustment

**Current value: `CONFIDENCE_GATE = 0.82f`**

The sweep output (printed to Gradle test stdout) shows whether a lower gate
(e.g. 0.78–0.80) would reduce FRR while keeping FAR near zero. Given:

1. The 63-dim geometry makes all FAR values effectively 0% across the 0.70–0.95
   sweep range (impostors are never close to the genuine centroid).
2. The dominant cost is FRR — false rejects degrade UX.
3. The spread-normalised confidence (Config B) provides adaptive per-user
   thresholding that is inherently more stable than raw cosine.

**Recommendation:** After reviewing the `CONFIDENCE_GATE sensitivity sweep`
test output, consider lowering `CONFIDENCE_GATE` to **0.78** if:
- Observed FRR > 25% at gate=0.82 across multiple real users
- FAR remains ≤ 1% at gate=0.78 (virtually guaranteed by 63-dim geometry)

This change requires editing one constant in `GestureClassifier.kt`:
```kotlin
const val CONFIDENCE_GATE = 0.78f  // was 0.82f; see docs/GESTURE_CLASSIFIER_AB_TEST.md
```

No other code changes are needed — `predict()` uses `CONFIDENCE_GATE` directly.

## Config B Advantages Over Config A

| Property | Cosine-only (A) | Full classifier (B) |
|----------|-----------------|---------------------|
| Adaptation to per-user gesture variability | None | Yes — spread computed from enrolment samples |
| Sensitivity to gesture change over time | High | Lower — re-training updates spread |
| FAR against random impostors | ~0% | ~0% |
| FRR at noise σ=0.15 | ~20–50% | ~10–35% |
| FRR at noise σ=0.40 (high variability) | ~60–80% | ~40–65% |
| Implementation complexity | Trivial | Low |
| Stored model size | 0 bytes | 256 bytes (63 floats + spread) |

**Winner: Config B** — equal security, lower rejection rate for genuine users.
